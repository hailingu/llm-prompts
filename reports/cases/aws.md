# AWS — 深度案例（补充）

## 概述
- 核心组件：S3（对象）、EBS（块）、EFS/FSx（文件）、Nitro/NVMe 实例、Elastic Fabric 等。  
- 典型场景：高并发数据库、AI训练（NVMe-oF）、大规模对象归档（S3 + Glacier）。

## 架构要点与工程实践
- 分层存储：S3（冷/热）+ EBS（性能分级）+ 本地 NVMe 缓存，用以权衡成本与性能。  
- 快照与备份：EBS 快照用于快速恢复与版本管理，S3 用于长期归档与生命周期管理。  
- 网络与并行：NVMe-oF / RDMA 或高带宽 ENI 用于高并发训练场景。

## 性能/成本估算（见：`reports/data/case_aws_estimates.csv`）
- 示例：高并发 OLTP（io2 + NVMe instances）可达到数十万 IOPS（取决于实例/卷类型与配置），AI 训练通过 NVMe-oF 可扩展到多 GB/s。
- 注：具体数值依赖实例类型、EBS 类型、网络分配与并行度，建议在试点中逐步调优。

## 试点清单（建议）
